	## URI's

<*> Text before URI - must be space|quote|bracket|slash (think what else)

<*> Should be treated as URI
	SYNTAX: scheme:[//[user[:password]@]host[:port]][/path][?query][#fragment]
	> see https://en.wikipedia.org/wiki/URI

	pre.name.com
	pre.website.co
	pre.pre2.pre3.website.co.uk
	x.com
	x.co
	name.com/sub/page
	name.com/sub/page/
	ab.co
	abc.com
	例子.卷筒纸
	http://example.com
	https://www.website.com/
	http://name.com// // Include second slash
	http://a_.aa/
	abc+.-://name.com
	text://name.com // Protocols can only be http, https and ftp (not random)
	ftp://public.ftp-servers.example.com/mydirectory/myfile.txt
	example.com/file.7zip // Include numbers in extensions
	example.com/file.a4
	example.com./ // Is this valid?
	http://例子.卷筒纸
	10fastfingers.com // Numbers sometimes break syntax
	www.10fastfingers.com
	1abc://name.com // Scheme must start with ASCII letter
	'example.com'
	(example.com)
	[example.com]
	{example.com}

<*> Should not be treated as URI
	x..
	~..
	we.c3% // Ending must be letters only
	a.b a.b. // 2+ ASCII letters in last part
	*%-:'.com // First character in name must be ASCII letter or number
	(.com
	^*:~.%-|'
	www.*%-:'.com
	website.ć
	name.com a // Formatting should end before a space
	https://w.c
	http://
	S.W.A.T.
	NUMBERS 123.456
	DATES 23.12.2017
	DATES 10/03/2018

<*> Punctuation after URI should not be formatted, except forward slash
	www.website.com.
	www.website.com...
	www.website.com!
	www.website.com?
	www.website.com^
